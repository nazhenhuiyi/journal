# 回声每日策展规则

## 页面分工

- 回声页每天只承载一条策展结果，不附带历史列表。
- 组件页先作为开发环境里的卡片样张库，之后可以从用户侧隐藏。
- 卡片不是堆叠展示，而是服务一个展品叙事：来源、摘录、策展人旁白、今天翻到它的理由和一个轻问题。

## 每日策展数据

- 每个本地日期只保存一个 `DailyCuration` 策展包。
- 重新生成会覆盖当天保存的策展数据，当前用于体验，之后可以下线。
- 每条策展必须保留来源日期、标题或摘录、标签、集合和可选图片，保证可追溯。
- AI 可以改写标题、旁白、理由和问题，但不能新增日记事实。

```ts
type DailyCuration = {
  version: 4
  today: TodayContext
  thesis: CurationThesis
  hero: EchoArtifact
  supports: EchoSupportCard[]
  closingQuestion: string
}
```

- `today` 负责固定“今天”的日期、星期、季节、天气、地点和今日日记摘要；天气只作为页眉背景与召回加权，不默认进入正文卡片。
- `thesis` 是这次策展的总判断：今天为什么适合翻到这一组旧页。
- `hero` 是唯一主展品，负责第一视觉和主要摘录。
- `supports` 是 2-3 张辅助卡片，用主题便签、旁证旧页、小票等方式让策展成立。
- `closingQuestion` 是最后留给用户的一个轻问题。

## 召回原则

- 优先根据时间节点相似性召回历史日记，例如同月、近似往年今日、季节相邻。
- 默认优先选择至少相隔 21 天的历史日记；如果日记库还不够远，则选择至少相隔 7 天的短程回声；再不够时才使用最近页。
- 再参考标签、集合、字数、图片和碎碎念数量，选择更适合被展示的一页。
- 今日天气、今日标签、今日日记摘要会参与召回加权。天气是环境背景，不默认成为策展解释。
- 不召回今天，也不从空白日记生成策展。
- 重新生成时可以在前几个高分候选之间切换，但仍然只保存一个今日策展包。

## 策展人口吻

- 像朋友，不像总结机器。
- 可以有轻微荒诞感，但不要把用户写过的东西说成事实之外的剧情。
- 推荐结构：来源日期与召回标签、原文摘录、策展人旁白、为什么今天翻到它、一个轻问题。
- 不向用户展示分数、版本、表格化线索或过重的算法说明。
- 页面可以展示多张卡，但每张卡必须有叙事角色：主题便签、旁证旧页、主题小票、对照记忆等。

## AI 接入边界

- 首版页面使用本地规则生成并保存，避免打开页面时自动触发 AI。
- 下一步 AI 接入可以只处理候选集：前端把 3-5 条召回候选、今日日期和口吻规则交给 AI。
- AI 输出 JSON，并由前端校验来源日期必须来自候选集。
- AI 失败时回退到本地规则生成。
