# 明信片情绪图案层素材流程

这份文档记录明信片「情绪图案层」的设计与生成流程。目标是沉淀一套可复用、可维护的透明 PNG 素材，让明信片在不影响正文阅读的前提下，多一层地点和心情的表达。

## 设计目标

- 用作明信片背景上的轻量装饰层，不作为主体插画。
- 画面要像旧纸上的淡印痕、压花、票根或水彩痕迹，而不是贴纸。
- 可爱感来自生活细节和手作感，不走高饱和、强卡通路线。
- 素材之间保持统一尺寸和画面密度，便于后续按 `tone` 维护。

## 统一规格

- 尺寸：`1024 x 1024`
- 格式：PNG
- 背景：透明 alpha
- 构图：低密度、留白多，主体尽量靠右侧、下侧或角落
- 使用方式：叠在明信片纸张层之上、正文内容之下
- 推荐 CSS：

```css
.journal-postcard-motif {
  position: absolute;
  right: -1.55rem;
  bottom: -2rem;
  z-index: 0;
  width: 18rem;
  height: 18rem;
  object-fit: contain;
  opacity: 0.3;
  pointer-events: none;
  mix-blend-mode: multiply;
  filter: saturate(0.82) contrast(0.96) sepia(0.08);
}
```

## 透明底验收

生成后必须检查文件是不是实际 alpha 透明，而不是画了棋盘格的假透明。

```bash
sips -g pixelWidth -g pixelHeight -g hasAlpha src/assets/postcard-motifs/river-light.png
sips -g pixelWidth -g pixelHeight -g hasAlpha src/assets/postcard-motifs/bookshop-ticket.png
```

期望结果：

```text
pixelWidth: 1024
pixelHeight: 1024
hasAlpha: yes
```

本次实际情况：提示词要求了透明背景，但生成结果最初是 `1254 x 1254` 的 RGB 图片，没有 alpha，并且把透明背景画成了棋盘格。因此最终资产经过了本地规范化处理：

- 从原始生成图重新导出，保留原图不删除。
- 缩放到 `1024 x 1024`。
- 去除棋盘格背景色，写出真实 alpha PNG。
- 再次用 `sips` 验证尺寸和 alpha。

处理命令参考：

```bash
magick <source.png> \
  -resize 1024x1024 \
  -alpha set \
  -fuzz 4% \
  -transparent '#ffffff' \
  -transparent '#fefefe' \
  -transparent '#efeeee' \
  -transparent '#ededed' \
  -transparent '#e4e4e3' \
  -transparent '#dddddd' \
  <target.png>
```

如果之后生成器直接给出真实透明 PNG，就不需要这一步，只保留尺寸与 alpha 验收。

## 接入方式

在明信片数据里给每张卡增加 `motifImage`：

```ts
{
  place: '苏州河边',
  tone: 'river',
  motifImage: riverMotifImage,
}
```

组件里将素材作为独立背景层插入：

```tsx
<article className={`journal-postcard is-${postcard.tone}`}>
  <img alt="" aria-hidden="true" className="journal-postcard-motif" src={postcard.motifImage} />
  ...
</article>
```

这样后续新增主题时，只需要：

1. 生成一张同规格 PNG。
2. 放到 `src/assets/postcard-motifs/`。
3. 在数据里绑定到对应 `tone`。
4. 如有必要，为特定 `tone` 微调位置和透明度。
