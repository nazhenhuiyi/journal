# 四季涂鸦素材生成流程

本文记录四季贴纸素材从风格探索、生成、切图到可用资产落地的经验。这里只讨论 PNG 贴纸和精灵图素材，不包含图标系统。

## 目标

四季素材用于日记产品里的情绪装饰、卡片角标、便利贴/明信片点缀和回忆页氛围。它们应该像画在纸页边角的小符号，而不是完整插画或商业图库贴纸。

第一版可采用 `4 x 6` 的精灵图结构：

- 春：嫩芽、小花、雨滴、微风、蝴蝶、绿枝
- 夏：太阳、冰饮、海浪、西瓜、萤火、小风扇
- 秋：枫叶、月亮、栗子、热茶、信封、稻穗
- 冬：雪花、围巾、蜡烛、热可可、星星、霜窗

每个素材最终输出为独立 `256 x 256` PNG，并保留一张总精灵图和一份 manifest。

## 风格原则

推荐风格是“产品化的手绘涂鸦贴纸”：

- 粗蜡笔、油画棒、干笔触颗粒。
- 边缘有不规则和漏白，但主体轮廓要清楚。
- 可爱但不过度儿童化，避免过甜、过圆、过卡通。
- 形状要符号化，能在较小 UI 尺寸下辨认。
- 色彩可以鲜明，但要统一在一套有限调色盘内。

推荐调色：

- cobalt blue
- deep leaf green
- strawberry pink
- golden yellow
- warm orange-red
- charcoal black
- soft cream accents

应避免：

- 写实照片感。
- 3D、玻璃质感、阴影、渐变、高光。
- 带标签、文字、数字或水印。
- 每个素材复杂度差异过大。
- 主体外散落太多小装饰，导致后续裁切困难。

## 生成提示词模板

生成时优先要求“一张精灵图”，但不要相信生成结果一定精确对齐。提示词重点是让每个对象独立、可切分、视觉权重接近。

```text
Use case: stylized-concept
Asset type: product-ready seasonal sticker sprite sheet for a desktop journal app
Primary request: Create a cohesive sprite sheet of 24 cute seasonal doodle stickers: spring, summer, autumn, winter.
Canvas: one sprite sheet, 4 rows x 6 columns, each asset centered in a consistent invisible 256x256 cell, generous padding, no visible grid, transparent background if supported.
Subjects: Spring row: sprout, simple flower, raindrops, breeze swirl, butterfly, leafy branch. Summer row: sun, cold drink, wave, watermelon slice, firefly, small fan. Autumn row: maple leaf, crescent moon, chestnut, tea cup, envelope, rice stalk. Winter row: snowflake, scarf, candle, hot cocoa mug, star cluster, frosty window symbol.
Style/medium: brand-like hand-drawn crayon symbols, chunky oil pastel strokes, dry wax texture, rough edges, imperfect but confident marks. Less childish, more tasteful diary-product doodle language.
Composition/framing: each sticker should be symbolic and readable at small UI sizes, simple silhouette, roughly equal visual weight, no tiny decorative extras outside the main shape, no labels.
Color palette: warm but clean crayon palette with cobalt blue, deep leaf green, strawberry pink, golden yellow, warm orange-red, charcoal black, soft cream accents.
Text: no text, no letters, no numbers, no labels.
Constraints: suitable for later slicing into sprites; each object isolated with crisp boundaries; no shadows, no paper texture background, no 3D, no glossy vector style, no photorealism, no watermark.
```

## Review 标准

生成后先做人工 review，不要直接接入项目：

- 气质：是否像日记边角的小贴纸，而不是图库插画。
- 统一性：四季之间是否同属一套笔触、色彩和复杂度。
- 可读性：缩小到 UI 小尺寸后是否还能认出来。
- 可切分性：每个素材之间是否有足够留白。
- 背景：是否真的透明；若是画出来的棋盘格或奶油底，需要重新抠除。
- 对齐：主体是否落在自己的网格中心。生成图经常看似 `4 x 6`，但实际主体会偏移。

若发现以下问题，建议再生成或进入后处理：

- 大量素材贴边或跨格。
- 某些主体过大，某些过小。
- 有文字、标签、数字、水印。
- 浅色主体和背景粘连严重。
- 小装饰太多，导致裁切框不可控。

## 文件组织

当前项目建议放在：

```text
src/assets/seasonal-doodles/
  season-doodle-sprites-v2.png
  season-doodle-sprites-v2-source.png
  season-doodle-sprites-v2.json
  spring/
    sprout.png
    flower.png
    raindrops.png
    breeze.png
    butterfly.png
    leafy-branch.png
  summer/
  autumn/
  winter/
```

建议保留：

- `*-source.png`：原始生成图，方便重切。
- `*.json`：精灵图坐标、名称和中文标签。
- 独立 PNG：实际使用最方便。
- QA 预览图：可选，用于检查透明和对齐，不一定进入最终打包。

## 后处理流程

### 1. 检查尺寸和 alpha

生成图可能声称透明，但实际是 RGB 或画了棋盘格。先检查：

```bash
sips -g pixelWidth -g pixelHeight -g hasAlpha path/to/generated.png
```

如果 `hasAlpha: no`，需要做背景抠除。如果图中是假的棋盘格，需要把接近白/灰的中性背景移除。

### 2. 背景抠除

可靠做法是从整张图边缘开始 flood fill，只移除与边缘连通的浅色中性背景。这样可以尽量保留信封、蜡烛、杯子等浅色主体内部。

关键经验：

- 不要简单把所有接近白色的像素都设透明，否则浅色主体会被吃掉。
- 从边缘连通背景开始抠除，比全局阈值更安全。
- 抠除后要在深色底上预览，白边和漏抠会更明显。

### 3. 不要机械按网格切

即使提示词要求 `4 x 6`、每格 `256 x 256`，生成结果里的主体也可能偏移。第一次切图后应检测每张图的 alpha bounding box：

```text
center offset = bbox center - 128
```

如果偏移达到十几甚至几十像素，就需要回到原始大图重新裁主体，而不是继续使用已经偏掉的小图。

### 4. 重新居中

更稳的流程：

1. 对原始大图做透明化。
2. 为每个素材设置一个宽松搜索窗口。
3. 在窗口内用 alpha bounding box 找到真实主体。
4. 裁出主体。
5. 必要时把最大边缩到约 `220px`，保留安全边距。
6. 粘贴回新的 `256 x 256` 透明画布中心。
7. 用所有新画布重组总精灵图。

这一步可以把主体中心偏移控制在约 `1px` 内。

### 5. 生成 QA 图

建议输出两类预览：

- 深色背景预览：检查透明边缘、白边、漏抠。
- 网格预览：检查每个 `256 x 256` 单元内的视觉中心。

网格预览只是 QA 文件，不应该作为正式素材使用。

## Manifest 格式

推荐 manifest：

```json
{
  "source": "./season-doodle-sprites-v2.png",
  "sourceOriginal": "./season-doodle-sprites-v2-source.png",
  "cellSize": 256,
  "columns": 6,
  "rows": 4,
  "sprites": [
    {
      "id": "spring.sprout",
      "season": "spring",
      "seasonLabel": "春",
      "name": "sprout",
      "label": "嫩芽",
      "file": "./spring/sprout.png",
      "x": 0,
      "y": 0,
      "width": 256,
      "height": 256
    }
  ]
}
```

命名规则：

- `id` 用 `<season>.<name>`。
- `file` 使用相对 `seasonal-doodles/` 的路径。
- 中文 `label` 用于编辑器、素材选择器或无障碍文本。
- `x/y/width/height` 对应总精灵图坐标。

## 质量门槛

素材进入产品前至少满足：

- 每个独立 PNG 为 `256 x 256`。
- 每个独立 PNG 带 alpha。
- 总精灵图带 alpha。
- 主体视觉中心基本位于画布中心。
- 主体最大边不贴边，至少保留约 `18px` 安全边距。
- 深色底预览无明显白边或背景残留。
- 同一季节内的素材视觉权重接近。
- 四季之间风格统一。

## 后续建议

如果要继续扩展，可以按“主题包”而不是单个元素零散生成：

- 天气包：晴、雨、雪、风、雾、雷。
- 情绪包：安静、开心、低落、想念、灵感、疲惫。
- 场景包：窗边、书桌、灯、咖啡、车票、花。

每个主题包仍沿用同样流程：先生成整张探索图，再 review，再透明化、重切、居中、生成 manifest。
