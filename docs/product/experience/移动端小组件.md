# 移动端小组件

这份文档只记录移动端小组件的产品形态和端侧实现。内容推荐规则见 [回顾与小组件推荐逻辑](回顾与小组件推荐逻辑.md)。

## 我们做了什么

移动端拆成两个用户可见的小组件：

| 小组件 | 尺寸 | 用途 | 内容来源 | 点击 |
| --- | --- | --- | --- | --- |
| 回看 | Medium | 展示回顾内容 | `bundle.review` | 周回顾、每日回顾或写作入口 |
| 此刻 | Compact | 快速记录入口 | `bundle.moment` | 对应 theme 的 murmur 输入 |

两者共享 `JournalWidgetBundleSnapshot` v2，但展示语义不同：

- `review`：`weekly-review`、`daily-review` 或 `empty-review`。
- `moment`：固定是 `theme-entry`。

snapshot 写在移动端本地 `journal-widget-snapshot-v2.json`，不进入 Git 同步。缺少 v2 时会读取旧 `journal-widget-snapshot-v1.json` 并适配成 v2。

## 怎么生成数据

刷新链路：

```txt
journalEffects
  -> refreshJournalWidgetSnapshot()
  -> listDailyJournals()
  -> listWeeklyReviews()
  -> loadOrCreateDailyReview()
  -> createJournalWidgetBundleSnapshot()
  -> 写 journal-widget-snapshot-v2.json
  -> updateNativeJournalWidgets()
```

刷新失败只记录错误，不阻断保存或同步。小组件会继续显示上一次有效 snapshot 或 fallback 文案。

## 回看怎么展示

回看只展示一条内容：

1. 7 天新鲜期内的最新周回顾。
2. 每日回顾 moment。
3. 空占位，点击进入 `small-thing`。

封面规则：

- `backgroundImageSrc` 暂时只作为 snapshot 数据保留。
- 小组件 MVP 不使用封面或图片背景，统一显示纯文字卡片。
- title 最多 2 行，summary 最多 3 行。

## 此刻怎么展示

此刻只做记录入口：

- 显示 theme title 和一句 subtitle。
- 同样不使用图片背景。
- 按本地时间段挑 theme，并避开今天已经用过的同一 theme。
- 点击整张卡片进入对应 theme 的 murmur 输入。

## Deep Link

小组件点击只走 deep link：

```txt
journal://weekly-review?week=<YYYY-Www>
journal://review-day?date=<YYYY-MM-DD>
journal://write?theme=<themeId>
journal://review
```

`App.tsx` 负责解析链接并整理导航栈。

## Android

Android 使用 `react-native-android-widget`。为了兼容旧桌面实例，provider 内部名不改，只改用户可见名称和渲染内容。

| Provider | 用户可见名 | 渲染 |
| --- | --- | --- |
| `JournalMoment` | 回看 | `bundle.review` |
| `JournalMomentCompact` | 此刻 | `bundle.moment` |

关键文件：

- `apps/mobile/src/widgets/JournalMomentAndroidWidget.tsx`
- `apps/mobile/src/widgets/widgetTaskHandler.tsx`
- `apps/mobile/src/widgets/journalWidgetNative.android.tsx`
- `apps/mobile/app.json`

主动刷新时先用 `getWidgetInfo()` 找到已添加实例，再用 `requestWidgetUpdateById()` 逐个刷新。

## iOS

iOS 使用 Expo SDK 56 的 `expo-widgets`。

| Widget kind | 用户可见名 | 尺寸 | 渲染 |
| --- | --- | --- | --- |
| `JournalMomentWidget` | 回看 | `systemMedium` | `bundle.review` |
| `JournalMomentCompactWidget` | 此刻 | `systemSmall` | `bundle.moment` |

关键文件：

- `apps/mobile/ios/ExpoWidgetsTarget/JournalMomentWidget.swift`
- `apps/mobile/ios/ExpoWidgetsTarget/JournalMomentCompactWidget.swift`
- `apps/mobile/src/widgets/JournalMomentCompactWidget.ios.tsx`
- `apps/mobile/src/widgets/journalWidgetNative.ios.ts`
- `apps/mobile/app.json`

WidgetKit 缓存较强，改原生 target 或 `app.json` 后需要重新 development build / EAS build。

## 验收

通用：

- 添加列表里能看到「回看」和「此刻」。
- 保存日记后 snapshot 更新，小组件能刷新。
- 点击能打开对应页面或输入入口。
- 没有回顾内容时，「回看」显示占位。

Android：

- 小米 / HyperOS 上验证添加、点击、保存后刷新。
- 「回看」周回顾、每日回顾、空占位都可读。
- 「此刻」Compact 不留大块空白，文字不早早省略。

iOS：

- 添加小组件预览和桌面都不出现红色错误卡。
- 「回看」周回顾、每日回顾、空占位都可读。
- 「此刻」能显示当前 theme 入口。
