# 移动端真机调试手册

这份文档沉淀移动端 Android 真机调试经验，尤其是 Expo / React Native development build、原生模块、键盘遮挡、同步和路由问题的排查流程。

## 先判断改动类型

JS / TS 层改动通常只需要 Metro reload。

当前移动端统一使用 development build 做真机调试和 E2E。这样键盘控制、Android widget、dev-client deep link、SecureStore 和原生配置都在同一条验收路径里。

需要重新构建 development build 的情况：

- 新增、升级或删除 native module，例如 `react-native-keyboard-controller`。
- 修改 `app.json` 里的 Android 原生配置、config plugin、权限、scheme、widget、manifest。
- 修改 `apps/mobile/plugins/*`。
- 修改 `android` / `ios` 原生工程，或 EAS build profile。

如果只刷新 JS 后出现 native module 缺失、原生配置不生效、widget 不更新等问题，先确认是否漏了 development build。

## 不覆盖正式包

真机里可能已经安装正式包 `app.zilin.journal`，它通常使用 EAS / release 签名。本地 debug 包使用 debug keystore，不能直接覆盖正式包。

常见安装错误：

- `INSTALL_FAILED_VERSION_DOWNGRADE`: 本地包 `versionCode` 低于手机已装版本。
- `INSTALL_FAILED_UPDATE_INCOMPATIBLE`: 包名相同但签名不同。

本项目本地 Android debug build 应使用单独包名：

```text
app.zilin.journal.debug
```

这样可以和正式包并存，避免覆盖真实日记数据。这个规则由 `apps/mobile/plugins/withAndroidDebugApplicationIdSuffix.js` 注入到 Android debug build。

## 真机连接

确认设备：

```sh
adb devices -l
```

如果有多台设备，记录 `serial`。例如：

```text
70eefe42 device model:25019PNF3C
```

USB 真机访问 Metro 推荐使用 `adb reverse`：

```sh
adb -s 70eefe42 reverse tcp:8081 tcp:8081
adb -s 70eefe42 reverse --list
```

然后使用 `127.0.0.1` 打开 development build：

```sh
adb -s 70eefe42 shell am start -W \
  -a android.intent.action.VIEW \
  -d 'exp+journal-mobile://expo-development-client/?url=http%3A%2F%2F127.0.0.1%3A8081' \
  app.zilin.journal.debug
```

如果 Expo 打开的地址是 `198.18.0.1` 或局域网 IP，真机不一定能访问。优先用 `adb reverse + 127.0.0.1` 排除网络因素。

## 构建和安装

本地 Android build 脚本会自动寻找 Java 17 和 Android SDK。默认候选包括：

- Homebrew `openjdk@17`
- Android Studio JBR
- `~/Library/Android/sdk`

运行本地 Android development build：

```sh
npm --workspace @journal/mobile run android:dev -- --device 25019PNF3C --app-id app.zilin.journal.debug --port 8081
```

如果 Expo 的 `--device` 传 adb serial 找不到设备，可以改用设备名称 / model，例如 `25019PNF3C`。

第一次 native build 会比较慢，后续 Gradle / CMake 会缓存。不要把本地生成的大型 `apps/mobile/android` 构建目录作为临时产物随手提交；需要长期保留的原生规则应沉淀成 config plugin。

## 日志和截图

清空日志：

```sh
adb -s 70eefe42 logcat -c
```

查看最近日志：

```sh
adb -s 70eefe42 logcat -d -t 500
```

过滤常见错误：

```sh
adb -s 70eefe42 logcat -d -t 800 | rg -i \
  'FATAL EXCEPTION|ReactNativeJS|Cannot find native module|Invariant Violation|journal-sync'
```

确认前台 Activity：

```sh
adb -s 70eefe42 shell dumpsys activity activities | rg \
  'topResumedActivity|mResumedActivity|app.zilin.journal'
```

截图：

```sh
adb -s 70eefe42 exec-out screencap -p > /tmp/journal-mobile-debug.png
```

截图适合确认键盘遮挡、路由前台状态、dev launcher 错误页、小组件实际尺寸。不要把包含 token 的 Maestro `commands-*.json` 原样输出到聊天或文档里。

## 键盘遮挡回归

长文输入遮挡不能只看页面是否整体变矮，还要确认光标所在行是否在键盘上方。

回归步骤：

1. 打开 `Today`。
2. 聚焦 `日记正文`。
3. 输入 40 行以上文本。
4. 继续输入最后一行，观察光标和最后一行是否在键盘上方。
5. 截图保存到 `/tmp`，不要提交截图。

示例输入：

```sh
for i in $(seq -w 1 45); do
  adb -s 70eefe42 shell input text "Line${i}_keyboard_visibility_test_content"
  adb -s 70eefe42 shell input keyevent ENTER
done
```

当前实现使用 `react-native-keyboard-controller`：

- 根节点挂 `KeyboardProvider`。
- Today 长文页使用 `KeyboardAwareScrollView`。
- Today 页关闭 `Screen` 自带的 `KeyboardAvoidingView`，避免两套键盘避让叠加。

## 路由和手势返回回归

小米 / HyperOS 上复现路由问题时，优先确认应用是否仍在前台，而不是只看“感觉退回桌面了”。

建议流程：

1. 从 Today 进入 Review。
2. 返回 Today。
3. 重复多次。
4. 通过 `dumpsys activity activities` 确认 `app.zilin.journal` 或 `app.zilin.journal.debug` 是否仍是 `topResumedActivity`。
5. 再用截图确认 UI 是否停在 Today。

如果测试 deep link 或 widget 跳转，返回 Today 应整理导航栈，避免历史栈把用户带到旧页面或桌面。

## 同步调试

同步问题先分层：

- UI 是否触发了保存 / 同步。
- SecureStore 是否有配置和 token。
- `@journal/sync` 是否报 Git 协议、merge、network、auth 错误。
- 真机网络是否能访问 GitHub。

日志过滤：

```sh
adb -s 70eefe42 logcat -d -t 1000 | rg -i 'journal-sync|remote|merge|push|fetch|github|ReactNativeJS'
```

同步 trace 的公共契约和后续接入边界见 `docs/product/同步 Trace 与日志接入.md`。

移动端 E2E 默认不跑真实 GitHub 同步。需要同步 flow 时显式设置 `JOURNAL_MOBILE_E2E_ENABLE_SYNC=1`，并只使用专用测试仓库 token。排查 Maestro 失败时优先看截图和 `maestro.log`，不要原样展示包含 token 的 commands JSON。

## 收尾清单

- 停掉 Metro / Expo server。
- 清理或移出临时 `apps/mobile/android` prebuild 目录，除非本次任务明确要提交 Android 原生工程。
- 保留必要 config plugin，让原生改动可复现。
- 跑 `npm --workspace @journal/mobile run typecheck`。
- 跑 `npm --workspace @journal/mobile run test`。
- 跑 `npm --workspace @journal/mobile run lint`。
- 跑 `git diff --check`。
