- p0 图片和碎碎念的存放与编写体验
  - 存放：当天长日记继续以 Markdown 文件为主，碎碎念作为 `:::murmur` block 写在当天文件底部；图片放在当天相邻的 media 目录中，正文只保留稳定相对路径和必要元数据。
  - 编写：长日记区域保持纯 Markdown 书写；碎碎念可以用轻量插入动作追加 block，默认带 `id`、`time` 和正文分隔线，用户只需要补文字。
  - 图片：插入图片时先复制/落盘到 media 目录，再向当前碎碎念追加 `::image` block，包含 `id`、`src`、可选 `caption` 和 `tags`。
  - 体验原则：源文件始终可读可迁移；UI 可以提供按钮和表单减少手写元数据，但不要生成只能由 App 理解的隐藏结构。

  - 预览支持三级有序列表，二级用 `a/b/c`，三级用 `i/ii/iii`。
