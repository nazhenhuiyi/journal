# 照片地图领域文档

这里放照片地图相关的产品、数据模型、交互、数据流和 MapLibre 技术说明。代码事实以移动端实现为准。

## 文档入口

- [照片地图功能总览](照片地图功能总览.md)：从产品视角解释照片地图是什么、屏幕上有哪些对象、图片和文字两条主线如何分工。
- [照片地图数据模型](照片地图数据模型.md)：解释上游输入边界、`PhotoMapMurmurSlice`、observation / cluster、视图状态和写回边界。
- [照片地图交互流转](照片地图交互流转.md)：按用户动作梳理 marker、卡片、sheet、tray、preview 和只读页之间的状态流转。
- [照片地图概念词典](照片地图概念词典.md)：统一 `observation`、`cluster`、`interaction`、`selectedTextId`、`tray`、`sheet` 等名词。
- [照片地图数据流与渲染](照片地图数据流与渲染.md)：实现层模块拆分、派生顺序、150m 聚合算法、渲染输入、视觉层级、相机和路线规则。
- [MapLibre 技术说明与依赖](<MapLibre 技术说明与依赖.md>)：MapLibre React Native 的组件模型、事件机制、Marker 与 Source/Layer 差异、依赖和调试边界。

## 推荐阅读顺序

1. 先读 [照片地图功能总览](照片地图功能总览.md)，建立产品心智。
2. 想确认对象关系、坐标来源和写回边界时，读 [照片地图数据模型](照片地图数据模型.md)。
3. 讨论某个点击、返回、展开、收起行为时，读 [照片地图交互流转](照片地图交互流转.md)。
4. 名词不确定时，查 [照片地图概念词典](照片地图概念词典.md)。
5. 要改代码或补测试时，读 [照片地图数据流与渲染](照片地图数据流与渲染.md)。
6. 遇到 marker 事件、地图空白点击、Source/Layer 取舍时，读 [MapLibre 技术说明与依赖](<MapLibre 技术说明与依赖.md>)。

## 维护边界

- 功能总览回答“这个功能对用户来说是什么”。
- 数据模型回答“照片地图自己的运行时对象有哪些、如何关联、是否会写回日记”。
- 交互流转回答“用户点某个地方后，状态应该怎么变”。
- 概念词典回答“这个名词具体指什么，不指什么”。
- 数据流文档回答“照片地图如何把日记内容变成地图体验，以及当前实现拆成哪些模块”。
- 技术文档回答“MapLibre RN 怎么用、有什么平台差异、事件为什么这样处理”。

## 避免重复维护

- 精确交互动作、状态流转、返回恢复和验收清单只维护在 [照片地图交互流转](照片地图交互流转.md)。
- 运行时模型字段、对象关系、坐标来源和写回边界只维护在 [照片地图数据模型](照片地图数据模型.md)。
- 派生顺序、聚合算法、渲染输入、视觉层级和相机规则只维护在 [照片地图数据流与渲染](照片地图数据流与渲染.md)。
- Marker、Source/Layer 和事件冒泡机制只维护在 [MapLibre 技术说明与依赖](<MapLibre 技术说明与依赖.md>)。
