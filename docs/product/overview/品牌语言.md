# 且留品牌语言

这份文档只维护品牌命名、语气和具体写法。产品定位、能力地图和非目标见 [产品总览](产品总览.md)。

## 品牌名

**且留**

“且”有一点暂且、慢一点、不急着解释的意思；“留”是保存，也是安放。

完整名称可以使用 **且留此刻**。在需要第一次介绍、应用商店、官网首屏或品牌故事时，用“且留此刻”讲清楚产品；在日常界面、图标、短文案和口语传播里，可以使用更轻的“且留”。

## Slogan

把此刻，轻轻留下。

## 辅助文案

万物有迹，心事且留。

## 一句话解释

给以后的自己，留一点今天的痕迹。

## 品牌气质

- 安静，不催促。
- 具体，不讲大道理。
- 有温度，但不表演亲密。
- 承认空白，不把沉默看成失败。
- 留白、在场、流动、温柔、东方、克制、有手感。

## 文案原则

- 少用“完成、任务、打卡、坚持、目标”。
- 少说“你应该”，多说“可以先”。
- 少说抽象鼓励，多指向真实事物：天气、纸页、照片、旧日、桌面、窗边。
- 按钮尽量是动作：写一页、留一句、拍一张照片、回到今天。

## 常用表达

| 场景 | 推荐写法 | 避免写法 |
| --- | --- | --- |
| 记录入口 | 先留一句 | 开始完成今日记录 |
| 空白状态 | 今天还没有留下什么 | 你还没有坚持记录 |
| 图片记录 | 拍一张照片 | 上传素材 |
| 回看旧日 | 慢慢翻回来 | 生成复盘报告 |
| 私有同步 | 放进自己的私有仓库 | 云端托管你的数据 |

## 官网首页写法

官网首页只讲用户能感到的事情：一句话、照片、地点、日期、回看和只留给自己。

技术实现不要放在首页主叙事里。本地优先、私有仓库和 Git 同步可以出现，但要解释它们对用户意味着什么：记录先属于自己，需要跨设备时也能带走。
