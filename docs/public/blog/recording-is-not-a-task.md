## 我们先承认：很多日子没有形状

不是每天都有一个适合被总结的主题。更多时候，生活只是一些散落的东西：路边的光，一杯没喝完的茶，突然想到的一句话，或者手机里一张并不完美的照片。

如果一个日记应用总是要求你把这些东西整理成漂亮文章，记录就会慢慢变成压力。且留更关心的是，在那些还没形成意义的时刻，能不能先给它们一个位置。

## 一页、一句、一张照片，都算数

且留把记录拆成几种很轻的动作。你可以写一整页，也可以只留一句碎碎念；你可以认真描述一天，也可以只放一张照片，让它以后替你把那天带回来。

这种设计不是为了降低标准，而是为了避免让标准挡在生活前面。记录不一定要完整，才配被保存。很多珍贵的东西，本来就是不完整地经过我们。

## 同步不应该让日记变成黑盒

且留使用可读的 Markdown 数据结构，并通过私有仓库同步。我们希望日记首先属于写下它的人，而不是属于某个看不见的服务。

当数据路径清楚，记录会多一点安心。你知道文字在哪里，照片如何被引用，设备之间如何同步；技术退到后面，留下来的东西仍然可以被你理解和带走。

## 回顾不是催你进步

我们做回顾、小组件和照片地图，不是为了提醒你做得够不够好，而是让旧日偶尔浮上来。也许你会看到一年前的天气，某段路，某句当时随手写下的话。

很多时候，记录的意义不是回到过去，而是在流动的生活里确认：我确实来过，我曾经在场。且留想保留的，就是这种轻轻确认的能力。
